# SessionSwitch macOS App v1 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** The real SessionSwitch: a native macOS menu-bar app (per PRD.md) that discovers running interactive Claude Code CLI sessions, shows each one's project/model/state, and switches a session's model/effort via slash-command injection — styled in the chosen "Mono Glass" design (dark neutral glass, amber/cyan/red signal accents).

**Architecture:** Swift Package Manager executable (no .xcodeproj), AppKit shell (NSStatusItem, NSPanel, `.accessory` activation policy) with SwiftUI views hosted where custom UI is needed. Three-layer core: Discovery (process scan → tty/cwd/terminal-app), StateReader (versioned adapter over `~/.claude/projects/<munged>/…jsonl`), Injector (AppleScript to Terminal.app/iTerm2, queue + verify). One `SessionStore` observable combines them on a 2 s refresh. All pure logic is protocol-injected and unit-tested with `swift test`; GUI paths are smoke-tested via a `--smoke-test` flag that boots and exits.

**Tech Stack:** Swift 6.3 (toolchain verified), macOS 14+ target (host = macOS 26), AppKit + SwiftUI, Carbon `RegisterEventHotKey` (global hotkey without Accessibility), `NSAppleScript`/`osascript` (Automation permission prompts on first use), SMAppService (launch at login). Zero third-party dependencies.

## Global Constraints

- Zero third-party dependencies. `Package.swift` platform floor: `.macOS(.v14)`.
- Layout: `app/` directory at repo root holds the package (`app/Package.swift`, `app/Sources/SessionSwitch/…`, `app/Tests/SessionSwitchTests/…`). design-lab/ and docs/ untouched except README.
- Verified ground truth (from live probing this machine — build the adapter to THESE facts):
  - Interactive session = `claude` process with a controlling TTY (`ps -axo pid=,tty=,command=`, tty ≠ `??`). Processes with `--input-format stream-json` in argv are IDE-piped (no tty or irrelevant) → exclude.
  - cwd: `lsof -a -p PID -d cwd -Fn` → line starting `n`.
  - Terminal app: walk `ppid` chain (`ps -o ppid= -p`), match ancestor command against known set: `Terminal`, `iTerm2`, `Code`/`Code Helper` (VS Code), `idea`/JetBrains, `ghostty`, `WarpTerminal`, `wezterm`, `alacritty`, `kitty`; unknown → "Terminal (unknown)".
  - State dir: `~/.claude/projects/<munged>` where munged = absolute cwd with every `/`, space, and `.` replaced by `-` (verified: `/Volumes/ExtremePro/My Projects/SessionSwitch` → `-Volumes-ExtremePro-My-Projects-SessionSwitch`).
  - Newest `*.jsonl` in that dir = active session log; assistant lines carry `"model":"claude-…"`; `"cwd"` lines confirm the mapping; **effort is NOT recorded in the jsonl** → v1 shows effort as unknown (`—`) unless/until a source is found; `/effort` injection still supported (fire-and-forget, verified only by absence of error).
  - Busy heuristic: jsonl mtime < 10 s old → `working`, else `idle`. `awaiting-permission` not detectable in v1 → omit that state.
- Injection tiering (v1): AppleScript for Terminal.app (`do script "…" in tab whose tty is …` — writes to that tab's tty) and iTerm2 (`write text` on the session whose `tty` matches). Everything else (VS Code/JetBrains/unknown/no-tty) = read-only with reason string. Never inject while session `working` unless forced: queue and apply on next refresh tick that reports `idle` (FR-8).
- Mono Glass theme tokens (from `design-lab/variants/11-mono-glass/style.css`): bg `#0a0a0c`, panel `rgba(22,22,24,.55)`, ink `#f2f2f2`, dim `#9a9a9a`, idle-gray `#6e6e73`, hairline `white@14%`; accents ONLY amber `#ffb000` (awaiting/pending/attention), cyan `#00e5ff` (verified/active), red `#ff3b3b` (rejected/error). No other saturated color.
- Model catalog defaults (mirroring the lab, FR-11 dynamic refresh deferred to v1.1): Fable 5 (`claude-fable-5`, alias `fable`, efforts low/medium/high/max), Opus 4.8 (`claude-opus-4-8`, `opus`, low/medium/high), Sonnet 5 (`claude-sonnet-5`, `sonnet`, low/medium/high), Haiku 4.5 (`claude-haiku-4-5`, `haiku`, none).
- Every task: `cd app && swift build && swift test` green, plus `.build/debug/SessionSwitch --smoke-test` prints `SMOKE OK` exit 0 for tasks that touch the app shell.
- Commit messages end with `Co-Authored-By: Claude Fable 5 <noreply@anthropic.com>`. Branch: `app-v1` off `main`.
- FR-37 privacy: never read message content from jsonl beyond scanning for `"model":`/timestamps; no network calls anywhere.

### Task 6: Injector — AppleScript build, queue, verify

**Files:**
- Create: `app/Sources/SessionSwitch/Core/Injector.swift`
- Test: `app/Tests/SessionSwitchTests/InjectorTests.swift`

**Interfaces:**

```swift
enum InjectionTarget { case terminalApp(tty: String), iTerm2(tty: String) }   // built from SessionInfo
enum ScriptBuilder {
    static func script(for target: InjectionTarget, command: String) -> String
    // Terminal.app:
    //   tell application "Terminal"
    //     repeat with w in windows
    //       repeat with t in tabs of w
    //         if tty of t is "<tty>" then do script "<escaped>" in t
    //       end repeat
    //     end repeat
    //   end tell
    // iTerm2:
    //   tell application "iTerm2"
    //     repeat with w in windows
    //       repeat with tb in tabs of w
    //         repeat with s in sessions of tb
    //           if tty of s is "/dev/<tty>" then tell s to write text "<escaped>"
    //         end repeat
    //       end repeat
    //     end repeat
    //   end tell
    // escaping: backslash and double-quote escaped for AppleScript string literals
    static func slashCommand(model: ClaudeModel) -> String          // "/model <alias>"
    static func slashCommand(effort: String) -> String              // "/effort <level>"
}
@MainActor final class Injector {
    init(store: SessionStore, execute: @escaping (String) -> String? = Injector.osascript)
    // execute returns error string or nil; default impl shells `osascript -e <script>` via Process
    func requestModel(_ m: ClaudeModel, for s: SessionInfo)
    func requestEffort(_ level: String, for s: SessionInfo)
    func applyPreset(_ p: Preset, for s: SessionInfo)               // model first, then effort if non-nil (0.5 s apart)
    // behavior: readOnly -> onResult(.rejected(reason)); state == .working -> enqueue, sets session.pending,
    // drains on store.onChange when that session reports .idle; idle -> inject immediately, set pending,
    // then verify: poll store snapshots up to 5 s for model change (model requests only) ->
    // .verified / .unverified("state file unchanged"); effort requests resolve .assumed (no state source)
    var onResult: ((Int32, InjectionResult) -> Void)?
    enum InjectionResult: Equatable { case verified(String), assumed(String), unverified(String), rejected(String) }
}
```

Use short poll intervals injected for tests.
